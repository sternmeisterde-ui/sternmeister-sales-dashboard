### Два разных вопроса: что оценивает OKK vs что показывает дашборд

OKK-сервис ставит `prompt_type` **по стадии** lead'a в Kommo (status-based routing). Дашборд группирует строки **по `managers.line`** (role-based filter), и для линии 2 дополнительно различает Бератер 1 / Бератер 2 через `prompt_type`. Эти две модели разные — потому что:

- Семантика промптов = РОЛЬ менеджера, а не стадия. `d2_dovedenie` — поддерживающий звонок без ролевки (роль Кристины+Лихваря); `d2_berater` — ролевка-подготовка к 1-му термину (роль Алекперов/Болтова/Милевская/Николаева).
- Стадии Kommo пересекаются: Кристина (line=3) и Алекперов (line=2) оба звонят на «Конс перед термином ДЦ». Один статус → разные роли → разный промпт по факту.

Если фильтровать дашборд по `prompt_type`, звонок Кристины со стадии «Конс ДЦ» (оценён как `d2_berater` потому что Kommo-стадия так маркирует) попадёт в Бератер 1 вместо Доведения. Был именно такой баг — дыра 30.04→17.05 в Доведении.

### Линия 1 — Квалификатор

| Воронка | Стадии | Тип оценки | Критериев |
|---|---|---|---|
| Бух Гос | Все стадии | d2_qualifier | 40 |
| Medical Гос | Все стадии | d2_qualifier | 40 |

### Линия 2 — Бератер (Бух Бератер воронка)

OKK ставит `prompt_type` по стадии:

| Стадия в Kommo | prompt_type (OKK ставит) | Семантика промпта |
|---|---|---|
| Неразобранное | d2_berater | Первый контакт, ролевка |
| Принято от первой линии | d2_berater | Первый контакт, ролевка |
| Доведение | d2_dovedenie | Поддерживающий, без ролевки |
| Консультация перед термином ДЦ | d2_dovedenie* | Подготовка к 1-му термину |
| Консультация перед термином ДЦ проведена | d2_dovedenie* | Подготовка к 1-му термину |
| Термин ДЦ отменен/перенесен | d2_berater | Первый контакт, ролевка |
| Термин ДЦ состоялся | d2_berater2 | После 1-го термина, ролевка АА |
| Термин АА отменен/перенесен | d2_berater2 | После 1-го термина, ролевка АА |
| Консультация перед термином АА | d2_berater2 | Подготовка ко 2-му термину |
| Консультация перед термином АА проведена | d2_berater2 | Подготовка ко 2-му термину |
| На рассмотрении бератера | d2_berater2 | После 1-го термина |
| Отложенный старт | d2_berater2 | После 1-го термина |
| Апелляция | d2_berater2 | После 1-го термина |
| Гутшайн одобрен | d2_berater2 | После 1-го термина |
| Закрыто и не реализовано | d2_berater2 | После 1-го термина |

*По спеке OKK эти стадии должны идти в `d2_dovedenie`, по факту OKK-сервис на ряде звонков отдаёт `d2_berater` (status-based приоритеты в коммите OKK не совпадают со спекой — см. `/Users/user/okk/src/config/constants.ts`). Из-за этого фильтр дашборда переехал на `managers.line`.

Дашборд (как фильтрует табы):

| Таб в Аналитике | SQL filter | Зачем |
|---|---|---|
| Все | `managers.line` без фильтра | Показывает все звонки, funnel-row по managers.line |
| Квалификатор | `managers.line='1'` | line=1 эксклюзивно d2_qualifier |
| Бератер 1 (2a) | `managers.line='2' AND prompt_type='d2_berater'` | Подготовка ДЦ |
| Бератер 2 (2b) | `managers.line='2' AND prompt_type='d2_berater2'` | Подготовка АА + пост-термин |
| Доведение | `managers.line='3'` | Кристина+Лихварь — все их звонки независимо от стадии |

Звонки с NULL managers.line (manager не в master_managers — задержка синка / удалён) попадают только в «Все» / «Без линии» funnel, в специфичных табах не видны.

## Какие поля берутся из Kommo

| Поле | Откуда | Для чего |
|---|---|---|
| Контакт (имя, ID) | `/contacts?query=телефон` | Привязка к контакту |
| Сделка (ID, название) | Через контакт → leads | Привязка к сделке |
| Воронка | `lead.pipeline_id` | Определение отдела (R2/D2) |
| Стадия | `lead.status_id` | Определение типа оценки |
| Ответственный | `lead.responsible_user_id` | Привязка к менеджеру |
| Ссылка на CRM | `leads/detail/{leadId}` | Ссылка в дашборде и Telegram |
| Custom fields | Только для D2 | Доп. информация по клиенту |

### Поиск контакта по телефону

1. Ищем контакт по номеру → берём все его сделки
2. Если сделок нет → ищем сделки напрямую п�� номеру
3. Из всех сделок выбираем лучшую:
   - Активная в известной воронке (приоритет)
   - Любая в известной воронке
   - Любая активная
   - Последняя по дате
