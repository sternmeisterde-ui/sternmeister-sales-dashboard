# MCP SternMeister — установка в Claude Desktop

Подключи MCP-сервер компании к Claude Desktop за 2 минуты — будешь получать ответы по дашборду на естественном языке.

## Что это даёт

35 tools поверх SternMeister-данных:
- Менеджеры (поиск, профиль, сравнение, табель)
- Реальные звонки ОКК (качество, ошибки, аудит)
- AI-ролевки (тренировки, gap analysis к OKK)
- Дейли план-факт + рефузалы
- Looker (cohort, SLA, outliers)
- Активность (workload, timeline, events)
- Термин ДЦ/АА (B2G)
- Скрипты + Анализ звонков

Один токен видит всё (B2G + B2B). Read-only — ничего не сломаешь.

## Установка (3 шага)

### 1. Открой конфиг Claude Desktop

**macOS**: `~/Library/Application Support/Claude/claude_desktop_config.json`
**Windows**: `%APPDATA%\Claude\claude_desktop_config.json`

Если файла нет — создай. Если есть — открой в любом редакторе.

### 2. Добавь блок mcpServers

Если файл пустой:
```json
{
  "mcpServers": {
    "sternmeister": {
      "url": "https://mcp.sternmeister.online/mcp",
      "headers": {
        "Authorization": "Bearer sk-mcp-5791da27499fd6f4d10e0fee6900821f79a301aadca61a67"
      }
    }
  }
}
```

Если в файле уже есть `mcpServers` — добавь только запись `"sternmeister": {...}` внутрь существующего блока (не удаляй другие серверы).

### 3. Перезапусти Claude Desktop

Полностью закрой (Cmd+Q / Alt+F4) и снова открой. Claude увидит MCP-сервер — можно работать.

## Как пользоваться

Просто задавай вопросы на русском. Claude сам выбирает нужные tools.

**Примеры:**
- «Какая картина по B2G сейчас?»
- «Покажи топ-5 худших звонков на 2-й линии за неделю»
- «У кого OKK score упал в апреле?»
- «Сравни Машу и Петю»
- «Топ ошибок которые AI находит»
- «У кого SLA первого звонка хуже 15 минут?»
- «Скрипт для бератера B2G»
- «Что Татьяна делала вчера в 14:00?»
- «Какие у вас вообще есть инструменты?» (Claude расскажет про 35 tools)

## Если что-то не работает

1. **Tools не появились** → перезапуск Claude Desktop полностью (не reload)
2. **«invalid bearer token»** → токен устарел, попроси новый у админа
3. **Tool возвращает ошибку** → пришли скрин админу, проверим audit log
4. **Server недоступен** → проверь https://mcp.sternmeister.online/health (должно быть `{"status":"ok"}`)

## Безопасность

- Токен компанейский — не публикуй и не пересылай за пределы команды
- Никакие данные не модифицируются (read-only)
- Все запросы логируются в audit log на стороне сервера
- Если токен утёк — админ rotate'ит, в Discord придёт новый
