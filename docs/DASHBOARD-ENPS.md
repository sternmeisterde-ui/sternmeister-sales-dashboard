# Вкладка «eNPS» — анонимный пульс-опрос менеджеров (b2g)

Last updated: 2026-07-06

## Что это

Еженедельный анонимный опрос морально-психологического состояния менеджеров
Гос ОП. Менеджеры заполняют Typeform «eNPS Gov» (форма `ircbf2JU`, аккаунт
владельца — не наш): балл 0–10 «Как ты оцениваешь свое эмоциональное состояние
на этой неделе?» + два открытых вопроса («что поддерживает» / «что мешает»).
Интеграция Typeform пишет ответы в Google-таблицу владельца формы, дашборд
зеркалит её в D1.

**Анонимность — принципиальна.** В данных нет и не должно быть привязки к
`master_managers`; вкладка показывает только агрегаты по неделям и анонимные
цитаты.

## Поток данных

```
Typeform → Google Sheets (владелец формы) → синк (сервис-аккаунт, read-only)
        → D1 enps_responses → /api/enps → EnpsTab
```

- **Таблица:** `enps_responses` (D1). Natural key — `token` (Typeform response
  token, колонка E листа), апсерт идемпотентный. Миграция
  `drizzle/d1/0003_enps_responses.sql`.
- **Синк:** `src/lib/enps/sync.ts`. Каждый прогон читает лист целиком
  (объём ~10–15 ответов/нед) и апсертит. `Submitted At` в листе — UTC.
- **Sheets-клиент:** `src/lib/enps/sheets.ts` — сервис-аккаунт, JWT RS256 на
  `node:crypto` без внешних зависимостей, scope `spreadsheets.readonly`.
- **Обновление:** stale-while-revalidate — GET `/api/enps` дёргает фоновый
  синк не чаще раза в 30 мин. Плюс форс-endpoint
  `GET /api/analytics/sync/enps` (под `x-cron-secret`; префикс
  `/api/analytics/sync/` уже в whitelist middleware).

## Env (и в Dokploy, и в whitelist `docker-compose.yml`!)

| Var | Что |
|---|---|
| `GOOGLE_OAUTH_JSON` | Полный JSON ключа сервис-аккаунта (client_email + private_key). Не путать с OAuth-тройкой `GOOGLE_OAUTH_CLIENT_ID/SECRET/REFRESH_TOKEN` выгрузки на Drive — это разные механизмы. |
| `ENPS_SPREADSHEET_ID` | ID живой Google-таблицы eNPS |
| `ENPS_SHEET_RANGE` | Опционально; дефолт `'eNPS Gov'!A2:E` |

Таблица должна быть расшарена (Читатель) на `client_email` сервис-аккаунта.
Без доступа синк graceful-фейлится (`PERMISSION_DENIED` в лог), вкладка живёт
на последних загруженных данных.

## API

- `GET /api/enps?from=YYYY-MM-DD&to=YYYY-MM-DD` — admin-only. Агрегаты:
  `totals` (count/avg/eNPS/промоутеры/нейтралы/критики), `weeks[]`
  (берлинские ISO-недели), `distribution[]` (0..10), `responses[]`
  (анонимные цитаты). Классификация: 9–10 промоутеры, 7–8 нейтралы,
  0–6 критики; eNPS = %промоутеров − %критиков.
- `GET /api/analytics/sync/enps` — форс-синк (CRON_SECRET).

## UI (`src/components/EnpsTab.tsx`)

Намеренно минимальная (упрощена по фидбеку 2026-07-06): пресеты периода
(12 нед / 26 нед / всё время), график среднего балла по неделям (кол-во
ответов — в тултипе) и таблица ответов «дата · балл · что поддерживает ·
что мешает». Клик по неделе на графике фильтрует таблицу. Бэк отдаёт больше
агрегатов (eNPS, структура, распределение) — UI их сейчас не рендерит.
