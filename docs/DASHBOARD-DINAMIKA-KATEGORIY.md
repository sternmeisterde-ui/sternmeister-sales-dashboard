# Динамика категорий (`category_dynamics`)

Last updated: 2026-07-22

Вкладка Коммерсов (b2b, admin-only, в NAV под «Дейли»): динамика распределения
лидов по подпериодам + продажи и конверсии — **пять таблиц-измерений** одной
и той же выборки лидов:

1. **Категории** — CATEGORY A–E + «Без метки» (зеркалит рабочий Google Sheet
   «Конверсия по категориям»);
2. **START_DATE** — «когда планируете начать» (анкета сайта);
3. **INCOME** — доход (анкета);
4. **STATUS** — занятость (анкета);
5. **LANGUAGE_LEVEL** — уровень языка (анкета).

## Компоненты и API

| Что | Где |

| Запрос + корзины | `src/lib/category-dynamics/data.ts` (`DIM_BUCKETS`, `bucketExpr()` — SQL-нормализация) |

| Миграции | `drizzle/analytics/0033_leads_cohort_first_payment_fact.sql`, `0034_leads_cohort_questionnaire.sql` |

## Определения (сверено 1в1 с выгрузками Kommo: июнь 459/27, март 500/24)

**«Правильное количество лидов»** из `analytics.leads_cohort`:
- период — по дате СОЗДАНИЯ лида (`created_at`, Berlin);
- воронки: Бух Комм 10631243 / Мед Комм 13209983 (тумблер);
- исключён этап Incoming leads (81523499 Бух; 101858011 Мед) и `is_deleted`;
- причины закрытия: авторитетно ТОЛЬКО поле 876383 «Причина закрытия
  (Обязательное поле)» (`b2b_close_reason_enum_id`); исключаемые 6 причин —
  Спам/Неквал/Сотрудничество/3 дубля; текстовый `loss_reason` — фолбэк только
  при пустом enum (у старых лидов поля противоречат: loss_reason=Спам при
  обязательном «Игнор»). ⚠ У SLA сет ШИРЕ (+ Гос. клиент, Неправильный
  контакт, решение Рузанны 2026-07-20) и логика по OR — списки НАМЕРЕННО
  разные, не «синхронизировать».

**Категория** — CFV 866934 `CATEGORY` (text: A/B/C/D/E), зеркалится в
`leads_cohort.category`. «Без метки» = пустое поле.

**Продажа** — заполнена «Факт. Дата 1-го платежа» (CFV 888296, по точному
field_id → `leads_cohort.first_payment_fact_date`, миграция 0033). Когортно к
дате создания лида: платёж в июле у июньского лида = продажа июня.
⚠ НЕ путать с `first_payment_date` — тот смешивает план/факт (findByName с
алиасами; на нём сидят факты Дейли — см. memory `daily-payment-fact-check`).

## Поля анкеты (Kommo CFV, лид)

У каждого вопроса в аккаунте ДВА поля: textarea-дубль и text. Дубли
(869872 «START_DATE.», 869876, 869878) — МЁРТВЫЕ (0 заполнений на выборке
500 свежих лидов, 2026-07-21). Рабочие text-поля:

| Вопрос | field_id | Колонка leads_cohort |
|---|---|---|
| START_DATE | 869932 | `start_date_answer` |
| STATUS | 869936 | `status_answer` |
| INCOME | 869938 | `income_answer` |
| LANGUAGE_LEVEL | 869928 | `language_level` (была с 0019, Funnel) |

Хранится сырой текст; корзины нормализуются на чтении (`bucketExpr()` в
data.ts), потому что форматы исторически дрейфуют:

- **INCOME**: «До 2 000» / «До 2000 евро» / «До 2 000 €» → `lt2`; средние
  корзины ловятся по цифрам (`regexp_replace [^0-9]` = `20003000`/`30005000`);
  «Выше…» → `gt5`.
- **STATUS**: «муж/жена» и опечатка «мужжена» → одна корзина `spouse`;
  «Получаю пособие, не работаю» — реальный ответ, которого не было в исходном
  списке, по решению 2026-07-21 отдельная корзина `benefit`.
- **LANGUAGE_LEVEL**: короткие «B1» и длинные «B1 (Средний уровень) — …»,
  кириллические «А1»/«А2» → TRANSLATE(кириллица→латиница) + LEFT(2). ⚠ UPPER
  в C-locale не трогает не-ASCII — поэтому сначала TRANSLATE.
- Пустое/неожиданное значение → корзина `""` («Без ответа»).

## UI

- Excel-калька: группы колонок = подпериоды (гранулярность автоматом: год →
  месяцы, месяц → недели, неделя → дни; первая группа — «Итого»), внутри
  группы колонки корзин; строки: Всего лидов (merged), Лиды категории,
  % от общего, Продажи, Конверсия от общего, Конверсия категории.
- Пять таблиц стеком, общий период/воронка/зум. Полный текст ответа — в
  title заголовка колонки и в тултипе ячейки.
- Цвета: категории и ответы — dark-слоты валидированной категориальной
  палитры (dataviz-skill, surface #0f172a); LANGUAGE_LEVEL — порядковая синяя
  шкала (A1 светлый → C2 тёмный, уровни упорядочены); «Без метки/ответа» —
  нейтральный серый.
- Липкая первая колонка с подписями метрик (горизонтальный свайп).
- Пресеты Неделя/Месяц/Год с подсветкой; стрелки листают календарную единицу
  целиком; ручной выбор в календаре = произвольный период (пресет гаснет).
- Клик по заголовку колонки — зум внутрь периода, кнопка «Назад».
- KPI-плитки: лиды / продажи / конверсия (+дельты к предыдущему окну или к
  периоду B) и топ-категория (по CATEGORY).
- «Сравнить периоды»: таблица B каждого измерения под его таблицей A
  (колонки выравниваются).
- Данные при листании не мигают (stale-while-revalidate); кэш API 5 мин.

## Данные/ETL

Ничего нового не тянется: обычный `sync-leads` заполняет `category`,
`first_payment_fact_date` (по field_id 888296 — в аккаунте ДВА поля «Дата
1-го платежа», матчить по имени нельзя) и три `*_answer`-колонки. Бэкфиллы:
факт-даты — 2026-07-14 (386 лидов); ответы анкеты — 2026-07-22, скоуп
created_at ≥ 2026-01-01 Berlin (решение: старше не трогаем), ответы у ~3.4k
из ~7k лидов (анкета есть только у лидов с сайта).
