# Daily — Коммерция (B2B) — Спецификация

Источник: `иксели/дейли коммерция.xlsx`, лист **ТЗ по Daily дашборду** (A-E строки 1-118).

Ответы на уточнения от 2026-04-24:
1. Sub-tabs внутри страницы Daily: **[Показатели] [Продления] [Менеджеры]** — переключатель, как режимы «Месяц по дням / Год по месяцам».
2. Режим «Недели» = **недели выбранного месяца (4-5 недель, Mon-Sun, пересекающиеся с месяцем).**
3. Дефолты «N% по умолчанию» — **жёстко зафиксированы, НЕ редактируются** (8% QL2P, 35 сек avg wait, 65% дозвон, 25 мин SLA, 85% ОКК).
4. **Только «ОКК Мед 1»**, никакой «Мед 2». R70 в ТЗ — опечатка, трактуем как «ОКК Мед 1 факт».
5. «Менеджеров на линии факт» в Weekly/Monthly = **уникальные менеджеры, которые выходили на линию хотя бы один день в периоде** (из schedule).

Департамент B2B — воронки **Бух Комм (`10631243`)** и **Medical Admin Commercial (`13209983`)**. Roleplay DB = R1, OKK DB = R2. Команда `ruzanna`.

## 8. Kommo custom-field lookup

Поля берём по имени (`field_name`, без учёта регистра; пробелы нормализуются) из `lead.custom_fields_values`:

```
firstPaymentDate    → «Факт. Дата 1-го платежа» | «Фактическая дата 1-го платежа»
firstPaymentAmount  → «Сумма 1-го платежа»      | «Сумма первого платежа»
prepaymentDate      → «Дата предоплаты»
prepaymentAmount    → «Сумма предоплаты»
```

Даты — Unix timestamp (сек) или строка; нормализуем в сек.

## 9. Исключения для «Квал ком. лидов факт»

**Бух:** created_at ∈ period, НЕ (status = Incoming `81523499`), НЕ lost(loss_reason `~Неквал` или `~Спам`).
**Мед:** created_at ∈ period, НЕ lost(loss_reason `~Неквал` или `~Спам`) (Incoming не исключаем).

Loss reasons — `GET /api/v4/leads/loss_reasons`, кэшируем 10 минут.

## 10. Светофор (применяется только к ФАКТУ)

Фон клетки факта:
- `< 50%` плана — `red` (bg-red-500/15)
- `50–99%` — `yellow` (bg-yellow-500/15)
- `≥ 100%` — `green` (bg-green-500/15)

Если плана нет — без цвета.

---

## 11. План propagation Monthly → Daily/Weekly

Для метрик с «Monthly: ручной»:
- `daily value = Math.round(monthlyValue / daysInMonth)`
- `weekly value = sum(daily values of this week's days that fall in the month)` — то есть если в Daily план 20 → в неделе из 5 дней = 100.
