## 🔄 ТЕКУЩАЯ ЗАДАЧА: Вкладка «Дейли» — подключение реальных данных из Kommo CRM

### Общая архитектура

```
Kommo API  →  Cron-эндпоинт (раз в час)  →  Neon: daily_metrics
                                                    ↓
                                              Next.js API  →  Фронтенд (вкладка Дейли)
```

- **CRM**: Kommo (amoCRM), API v4
- **Частота синка**: раз в час
- **Хранение**: таблица `daily_metrics` в Neon (D1_roleplay)
- **Формулы**: считаются в sync job, фронтенд показывает готовые числа

### Шаг 3: Создать таблицу daily_metrics в Neon

- [ ] Создать таблицу:
  ```sql
  CREATE TABLE daily_metrics (
    id SERIAL PRIMARY KEY,
    department TEXT NOT NULL,        -- 'b2g' | 'b2b'
    section TEXT NOT NULL,           -- 'funnel' | 'qualifier' | 'second_line'
    metric_key TEXT NOT NULL,        -- 'total_leads_plan', 'qual_rate', ...
    metric_value TEXT NOT NULL,      -- значение (строка, чтобы хранить %, шт., мин.)
    period_date DATE NOT NULL,       -- за какую дату
    synced_at TIMESTAMPTZ DEFAULT NOW()
  );
  CREATE INDEX idx_daily_metrics_lookup ON daily_metrics(department, period_date, section);
  ```
- [ ] Создать Drizzle-схему для таблицы
